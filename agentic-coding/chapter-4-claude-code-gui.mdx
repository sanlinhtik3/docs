အရှေ့ Chapter ၃ မှာ Coding Agent တွေရဲ့ အတွင်းပိုင်း အလုပ်လုပ်ပုံ Primitives တွေကို လေ့လာခဲ့ပြီးပြီ။ ဒီ Chapter 4 မှာတော့ အဲဒီ အေးဂျင့်တွေကို လက်တွေ့ လုပ်ငန်းခွင်မှာ ဘယ်လို ထိထိရောက်ရောက် မောင်းနှင်မလဲ ဆိုတာကို ဆွေးနွေးသွားကြမယ်။

အရင်ကဆိုရင် Agentic Coding ကို သုံးဖို့အတွက် အလင်းရောင်မရှိတဲ့ Terminal မျက်နှာပြင် မည်းမည်းကြီးထဲမှာ Command Line Interface (CLI) ကနေတစ်ဆင့် စာသားတွေ ထိုင်ရိုက်ပြီး ညွှန်ကြားခဲ့ကြရတယ်။ ဒါပေမယ့် ပွင့်ပွင့်လင်းလင်း ပြောရရင် အခုခေတ်မှာ ဘယ် Developer မှ ရှုပ်ထွေးတဲ့ Command flags တွေကို အလွတ်ကျက်ပြီး Terminal ထဲမှာ အချိန်ကုန် မခံချင်ကြတော့ဘူး။

ဒါကြောင့် လက်ရှိ အချိန်မှာ စွမ်းဆောင်ရည်မြင့်မားတဲ့ AI Operator တွေဟာ ခေတ်ဟောင်း CLI ကို စွန့်ခွာပြီး **GUI Apps (Desktop Applications နဲ့ Graphical IDEs)** တွေဖြစ်တဲ့ Claude Desktop App၊ Cursor၊ Windsurf နဲ့ Antigravity IDE လို App တွေကို ပြောင်းလဲ အသုံးပြုလာကြပါတယ်။

## ဘာကြောင့် CLI ကို မသုံးဘဲ GUI App ကို ပြောင်းသုံးကြတာလဲ? (Why GUI over CLI?)

Terminal ထဲမှာ စာရိုက်ရတဲ့ CLI နဲ့ Graphic မျက်နှာပြင် ပါဝင်တဲ့ GUI App တို့ရဲ့ ကွာခြားချက်ကို သေချာ ယှဉ်ကြည့်လိုက်ရအောင်:

| အချက်အလက် | ခေတ်ဟောင်း CLI (Terminal-based) | ခေတ်သစ် GUI App (Desktop / IDE-based) |
| :--- | :--- | :--- |
| **အသုံးပြုမှု (Usability)** | Command တွေ၊ flag တွေကို အလွတ်ကျက်ပြီး စာရိုက်ရတယ်။ | Mouse နဲ့ Click နှိပ်ရုံပဲ၊ Drag & drop လုပ်ပြီး အလွယ်တကူ သုံးလို့ရတယ်။ |
| **ကုဒ်စစ်ဆေးမှု (Code Review)** | Text အနေနဲ့ပဲ ပေါ်လို့ ဘာတွေ ပြင်သွားလဲဆိုတာ ဖတ်ရခက်တယ်။ | Visual Diff (အနီ/အစိမ်း) နဲ့ ရှင်းရှင်းလင်းလင်း မြင်ရလို့ စက္ကန့်ပိုင်းအတွင်း စစ်ဆေးနိုင်တယ်။ |
| **Context ထည့်သွင်းမှု** | File path တွေကို manual ရိုက်ထည့်ရတယ်။ | `@file` သို့မဟုတ် folder ကို Drag ဆွဲထည့်ရုံနဲ့ Context window ထဲ ရောက်သွားတယ်။ |
| **လုံခြုံရေး (Safety Gates)** | Command တစ်ခု run တိုင်း terminal မှာ စာသားနဲ့ အတည်ပြုရတယ်။ | App ပေါ်မှာ 'Approve' / 'Reject' ခလုတ်လေးတွေ ပေါ်လာလို့ စိတ်ချလက်ချ ထိန်းချုပ်နိုင်တယ်။ |

---

## GUI App တွေမှာ အလုပ်လုပ်တဲ့ အဓိက အဆင့်များ (Core Workflow in GUI Apps)

GUI App တစ်ခုကို သုံးပြီး Coding Agent ကို မောင်းနှင်တဲ့အခါ အောက်ပါ အဆင့် ၄ ဆင့်အတိုင်း ချောမွေ့စွာ အလုပ်လုပ်နိုင်တယ်:

### ၁။ Workspace ဖွင့်ခြင်း (Opening your Workspace)
မင်း တည်ဆောက်မယ့် ပရောဂျက် Folder ကို App ထဲမှာ ဖွင့်လိုက်ရုံနဲ့ အေးဂျင့်က မင်းရဲ့ Codebase ဖွဲ့စည်းပုံတစ်ခုလုံးကို အလိုအလျောက် Scan ဖတ်ပြီး နားလည်သွားတယ်။ 

### ၂။ Visual Context ထည့်သွင်းခြင်း (Adding Context Visually)
Terminal မှာလို ဖိုင်လမ်းကြောင်းတွေ အရှည်ကြီး ရိုက်ထည့်စရာ မလိုတော့ဘူး။ Chat box ထဲမှာ `@` လေး ရိုက်ပြီး ကိုယ်ပြင်ချင်တဲ့ ဖိုင်ကို ရွေးထည့်လိုက်ရုံပဲ။ ထူးခြားတာက UI/UX ပြင်ချင်ရင် စာနဲ့ ရှင်းပြနေစရာ မလိုဘဲ ပုံ သို့မဟုတ် Screenshot ကို Drag ဆွဲပြီး App ထဲ ချလိုက်ရုံနဲ့ အေးဂျင့်က မျက်စိနဲ့ မြင်သလို တန်းသိနိုင်တယ်။

### ၃။ Plan ကို Visual Review လုပ်ခြင်း (Reviewing the Plan Visually)
အေးဂျင့်ကို Goal ပေးလိုက်တဲ့အခါ သူက ကုဒ်တန်းမရေးဘဲ ဘေးဘက် Sidebar မှာ Markdown Todo list နဲ့ အစီအစဉ် (Plan) ကို သပ်သပ်ရပ်ရပ် ချပြတယ်။ အဲဒီ အစီအစဉ်မှာ မလိုအပ်တဲ့ Over-engineering တွေ ပါနေလား ဆိုတာကို မင်းက မျက်စိနဲ့ တစ်ချက်ကြည့်ပြီး စိတ်ကြိုက်ဖြစ်မှ 'Proceed' ကို နှိပ်ပြီး ခွင့်ပြုပေးလို့ ရတယ်။

### ၄။ Visual Diff ကြည့်ပြီး အတည်ပြုခြင်း (Inspecting Diffs & Approving)
အေးဂျင့်က ကုဒ်တွေကို ပြင်ဆင်လိုက်တဲ့အခါ App မျက်နှာပြင်ပေါ်မှာ ဘာတွေ ဖျက်လိုက်တယ် (အနီရောင်)၊ ဘာတွေ အသစ်တိုးလိုက်တယ် (အစိမ်းရောင်) ဆိုတာကို ဘေးချင်းယှဉ် (Side-by-side Diff) ပြသပေးတယ်။ ကုဒ်အပြောင်းအလဲဟာ မှန်ကန်တိကျတယ် ဆိုတာ သေချာမှ 'Accept' ခလုတ်လေးကို Click နှိပ်ပြီး ကိုယ့်ရဲ့ Codebase ထဲ အပြီးသတ် ထည့်သွင်းလိုက်ရုံပါပဲ။

## GUI App တွေနဲ့ သုံးတဲ့အခါ ထိရောက်စေမယ့် Operator နည်းဗျူဟာများ (Best Practices for GUI Operators)

GUI App တွေက ဘယ်လောက်ပဲ သုံးရလွယ်ကူနေပါစေ၊ မင်းဘက်က စနစ်ကျတဲ့ **Operator Mindset** ရှိဖို့တော့ မဖြစ်မနေ လိုအပ်တယ်:

* **Screenshot တွေကို အပြည့်အဝ အသုံးချပါ:** Design လွဲနေတာ၊ Alignment မကျတာတွေကို စာနဲ့ အရှည်ကြီး ရှင်းပြမနေပါနဲ့။ Error တက်နေတဲ့ UI ကို Screenshot ရိုက်၊ ဝိုင်းပြပြီး App ထဲ Drag ဆွဲထည့်ကာ *"ဒီပုံထဲကအတိုင်း ဘေး margin ကပ်နေတာကို ပြင်ပေးပါ"* လို့ ခိုင်းပါ။ အချိန် ၈၀% ပိုသက်သာပြီး ပိုတိကျတယ်။
* **Sidebar နဲ့ Editor ကို တွဲကြည့်ပါ:** အေးဂျင့်က Sidebar မှာ စဉ်းစားပြီး အလုပ်လုပ်နေတဲ့ အချိန်မှာ ကိုယ့်ရဲ့ Main Editor မှာ ဖိုင်တွေ Real-time ပြောင်းလဲနေတာကို စောင့်ကြည့်ပါ။ ကိုယ် မခိုင်းထားတဲ့ ဖိုင်တွေကို အေးဂျင့်က လျှောက်ပြင်နေရင် 'Stop' သို့မဟုတ် 'Cancel' ခလုတ်ကို ချက်ချင်း နှိပ်ပြီး တားမြစ်ရမယ်။
* **၈၀/၂၀ စည်းမျဉ်းကို အမြဲ သတိရပါ:** App က သုံးရလွယ်တိုင်း မလိုအပ်တဲ့ library တွေ၊ feature အသစ်တွေကို အလွယ်တကူ လျှောက်ထည့်ခိုင်းတာမျိုး မလုပ်ပါနဲ့။ Core functions ၂၀% ကိုပဲ အာရုံစိုက်ပြီး ရှင်းရှင်းလင်းလင်း တည်ဆောက်ပါ။

## သတိကြီးစွာ ထိန်းကျောင်းရမယ့်အချက်တွေ (Risks & Caveats)

<Warning>
**၁။ GUI Illusion (လှပတဲ့ Interface ကို သာယာပြီး မျက်ကန်းယုံမိတာ):**
GUI App က ကုဒ်တွေကို အရောင်အသွေး စုံလင်စွာနဲ့ သပ်သပ်ရပ်ရပ် ပြသပေးတာကြောင့် အေးဂျင့်ရေးတာ အကုန်မှန်တယ်လို့ ထင်ယောင်ထင်မှား ဖြစ်တတ်တယ်။ ဘယ်လောက်ပဲ လှပနေပါစေ၊ ကိုယ်တိုင် Automated tests တွေ မစမ်းဘဲ၊ Browser မှာ လက်တွေ့ Run မကြည့်ဘဲ မျက်ကန်း မယုံကြည်ပါနဲ့။

**၂။ Accidental Clicks (မစဉ်းစားဘဲ Allow / Proceed ခလုတ် လျှောက်နှိပ်မိတာ):**
App ပေါ်မှာ Terminal command run ဖို့ သို့မဟုတ် ဖိုင်ဖျက်ဖို့ Permission တောင်းတဲ့ Dialog box ပေါ်လာတဲ့အခါ အကျင့်ပါပြီး 'Approve' ခလုတ်ကို မျက်စိမှိတ် မနှိပ်လိုက်ပါနဲ့။ `rm -rf` လို ပရောဂျက် တစ်ခုလုံး ဖျက်ဆီးမယ့် command တွေ ပါနေလား ဆိုတာကို စာကြောင်း သေချာ ဖတ်ပြီးမှ အတည်ပြုချက် ပေးပါ။
</Warning>

## သင်ခန်းစာ checklist (Lesson Checklist)

ဒီ Chapter 4 ကို ဖတ်ပြီးတဲ့နောက် မင်းအနေနဲ့ အောက်ပါအချက်တွေကို တကယ်ပဲ နားလည်သဘောပေါက်သွားပြီလား ကိုယ့်ကိုယ်ကိုယ် ပြန်စစ်ကြည့်ပါဦး:

- [ ] ခေတ်ဟောင်း CLI ထက်စာရင် GUI App တွေက Code review နဲ့ Context management မှာ ဘာကြောင့် ပိုထိရောက်လဲဆိုတာ နားလည်ပြီလား။
- [ ] Visual Diff (အနီ/အစိမ်း) ကို ကြည့်ပြီး အေးဂျင့် ပြင်သွားတဲ့ ကုဒ်တွေကို စက္ကန့်ပိုင်းအတွင်း စစ်ဆေးတတ်ပြီလား။
- [ ] UI/UX ပြဿနာတွေကို စာနဲ့ ရှင်းပြမယ့်အစား Screenshot ရိုက်ပြီး Drag & drop လုပ် ခိုင်းစေတဲ့ နည်းဗျူဟာကို အသုံးချတတ်ပြီလား။
- [ ] App ပေါ်က 'Approve' ခလုတ်ကို မျက်စိမှိတ် မနှိပ်ဘဲ Command နဲ့ Code changes တွေကို သေချာ ဖတ်ပြီးမှ အတည်ပြုတတ်ပြီလား။

## နောက်တစ်ဆင့် ဘာဆက်သွားမလဲ (What's next?)

အခုဆိုရင် မင်းဟာ Agentic Coding ကို ခေတ်သစ် GUI App တွေ၊ Desktop IDE တွေပေါ်မှာ ဘယ်လို စနစ်တကျနဲ့ မြန်မြန်ဆန်ဆန် မောင်းနှင်ရမလဲ ဆိုတာကို လက်တွေ့ ကျွမ်းကျင်သွားပါပြီ။

နောက်တစ်ဆင့်မှာတော့ ကိုယ့်ရဲ့ Local စက်မှာတင် မကဘဲ၊ စွမ်းဆောင်ရည် မြင့်မားတဲ့ Cloud GPU တွေ၊ Remote Server တွေပေါ်မှာ အေးဂျင့်ကို လှမ်းချိတ်ပြီး အလုပ်လုပ်မယ့် **[Chapter 5: Codex & Remote GPU Coding](/agentic-coding/chapter-5-codex-remote-gpu)** ကို ဆက်လက် လေ့လာသွားကြမယ်!
