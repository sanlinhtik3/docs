အရှေ့ Chapter 2 မှာ မင်းဟာ Vibe Coder ဘဝကနေ စနစ်ကျတဲ့ **AI Operator** တစ်ယောက်အဖြစ် စိတ်နေစိတ်ထား ပြောင်းလဲခဲ့ပြီးပြီ။ ဒီ Chapter 3 မှာတော့ ကိုယ်မောင်းနှင်မယ့် Coding Agent တွေရဲ့ စက်ဖုံးကို ဖွင့်ပြီး အတွင်းပိုင်းမှာ ဘယ်လို အလုပ်လုပ်နေလဲ (Under the Hood) ဆိုတဲ့ အဓိက သဘောတရားတွေကို လေ့လာသွားကြမယ်။

ကားတစ်စီးကို မောင်းတဲ့အခါ အင်ဂျင်အလုပ်လုပ်ပုံကို နားလည်ထားတဲ့ ယာဉ်မောင်းဟာ ကားထိုးရပ်သွားတဲ့အခါ ဘာကြောင့်လဲဆိုတာ ချက်ချင်းသိပြီး အလွယ်တကူ ပြင်ဆင်နိုင်သလိုပဲ၊ Coding Agent တွေရဲ့ အတွင်းပိုင်း မောင်းနှင်အား (Primitives) တွေကို နားလည်ထားမှ မင်းရဲ့ အေးဂျင့် လမ်းပျောက်တဲ့အခါ တိတိကျကျ ထိန်းကျောင်းနိုင်မှာပါ။

## ဘာလဲ (What are Agent Primitives?)

AI Coding Agent တစ်ခုဆိုတာ မှော်ပညာနဲ့ အလုပ်လုပ်နေတာ မဟုတ်ဘူး။ သူ့ရဲ့ အဓိက ဦးနှောက်ဖြစ်တဲ့ Large Language Model (LLM) ကို အောက်ပါ အခြေခံ မောင်းနှင်အား **Primitives ၃ ခု** နဲ့ စနစ်တကျ ချိတ်ဆက် ပေါင်းစပ်ထားတဲ့ ပတ်လမ်း (Loop) တစ်ခုပဲ ဖြစ်တယ်:

1. **Tools (ကိရိယာတွေ):** အေးဂျင့်က မင်းရဲ့ codebase နဲ့ terminal ကို လက်တွေ့ ထိတွေ့ကိုင်တွယ်တဲ့ လက်နဲ့ မျက်စိများ။
2. **Planning (အစီအစဉ်ချတာ):** ကုဒ်တန်းမရေးခင် ပြဿနာကို အဆင့်ဆင့် ခွဲခြမ်းစိတ်ဖြာပြီး ခြေလှမ်းတွေ ကြိုတင်စဉ်းစားတဲ့ စနစ်။
3. **State & Memory (မှတ်ဉာဏ်နဲ့ လက်ရှိအခြေအနေ):** မင်းနဲ့ ပြောထားတဲ့ စကားလမ်းကြောင်း (History) နဲ့ လက်ရှိ ပရောဂျက်ရဲ့ အပြောင်းအလဲတွေကို မှတ်သားထားတဲ့ နေရာ။

ဒီ Primitives ၃ ခု ဘယ်လို အချိတ်အဆက်မိမိ အလုပ်လုပ်လဲဆိုတာ တစ်ခုချင်းစီ အသေးစိတ် ကြည့်လိုက်ကြရအောင်။

## ၁။ Tools: အေးဂျင့်ရဲ့ လက်နဲ့ မျက်စိ (Hands and Eyes of the Agent)

ရိုးရိုး Chatbot (ဥပမာ - Web ChatGPT) ဟာ ဖန်ပုလင်းထဲ ပိတ်လှောင်ခံထားရတဲ့ ဦးနှောက်တစ်ခုလိုပဲ။ စကားတော့ အလွန်ပြောတတ်တယ်၊ ဒါပေမယ့် မင်းရဲ့ ကွန်ပျူတာထဲက ဖိုင်တွေကို မြင်လည်း မမြင်ရဘူး၊ ပြင်လည်း မပြင်နိုင်ဘူး။

**Autonomous Coding Agent (Claude Code / Codex)** တွေကတော့ အဲဒီလို မဟုတ်တော့ဘူး။ သူတို့ကို ကွန်ပျူတာထဲမှာ လက်တွေ့ အလုပ်လုပ်နိုင်စွမ်းပေးတဲ့ **Tools** တွေ တပ်ဆင်ပေးထားတယ်:

* **File Read / View:** Codebase ထဲက ဖိုင်တွေကို ဖွင့်ဖတ်ပြီး နားလည်အောင် ကြည့်တဲ့ မျက်စိ (ဥပမာ - `view_file` tool)။
* **File Write / Edit:** ကုဒ်အသစ်တွေ ရေးသားတာ၊ မလိုတာတွေ ဖျက်တာ၊ ပြင်ဆင်တာတွေကို လုပ်ဆောင်တဲ့ လက် (ဥပမာ - `replace_file_content` tool)။
* **Grep Search:** ဖိုင်ပေါင်း ရာထူးချီရှိတဲ့ ပရောဂျက်ကြီးတွေထဲမှာ ကိုယ်လိုချင်တဲ့ function သို့မဟုတ် variable ကို စက္ကန့်ပိုင်းအတွင်း ရှာဖွေပေးတဲ့ ကိရိယာ။
* **Terminal Execution:** Build commands တွေ run တာ၊ automated tests တွေ စမ်းသပ်တာနဲ့ git operations တွေကို လုပ်ဆောင်ပေးတဲ့ tool။

### Operator တစ်ယောက်အတွက် ဘာကြောင့် အရေးကြီးတာလဲ?
မင်းရဲ့ အေးဂျင့်မှာ ဘာ tools တွေ ရှိလဲဆိုတာ သေချာသိထားမှ မှန်ကန်တဲ့ ညွှန်ကြားချက် ပေးနိုင်မယ်။ ဥပမာ - ဖိုင် ၁၀၀ လုံးကို မျက်စိမှိတ် ဖွင့်ဖတ်ခိုင်းမယ့်အစား *"Grep search သုံးပြီး `loginHandler` ရှိတဲ့နေရာကို အရင်ရှာပါ"* လို့ ထိထိမိမိ ခိုင်းစေတတ်ရမယ်။

## ၂။ Planning: မလုပ်ခင် စဉ်းစားတဲ့ ဦးနှောက် (The Thinking Engine)

Vibe Coding မှာ အဖြစ်များဆုံး ပြဿနာက အေးဂျင့်ကို စကားတစ်ခွန်း ပြောလိုက်တာနဲ့ ဘာအစီအစဉ်မှ မရှိဘဲ ကုဒ်တွေကို ကောက်ရေးတော့တာပဲ။ အဲဒီအခါ တစ်နေရာကို ပြင်လိုက်ရင် အခြားတစ်နေရာမှာ သွားပျက်တဲ့ Spaghetti code တွေ ဖြစ်ကုန်ရော။

စွမ်းဆောင်ရည်မြင့်တဲ့ Coding Agent တွေမှာတော့ **Planning Phase** ဆိုတာ မဖြစ်မနေ ပါဝင်တယ်:

* **Step-by-step Breakdown:** မင်းပေးလိုက်တဲ့ Goal (ပန်းတိုင်) ကို မြင်တာနဲ့ တစ်ခါတည်း အပြီးမလုပ်ဘဲ အဆင့်သေးသေးလေးတွေ (Tasks) အဖြစ် အရင် ခွဲခြမ်းစိတ်ဖြာတယ်။
* **Risk Assessment:** ဒီကုဒ်ကို ပြင်လိုက်ရင် ဘယ်ဖိုင်တွေ ထိခိုက်နိုင်လဲဆိုတာ ကြိုတင် တွက်ဆပြီး `implementation_plan.md` လို အစီအစဉ်ဆွဲတယ်။
* **Human Review:** ဆွဲထားတဲ့ plan ကို မင်းဆီ တင်ပြပြီး အတည်ပြုချက် (Approval) ရမှ လက်တွေ့ ကုဒ်ပြင်တဲ့အဆင့် (Act) ကို ကူးပြောင်းတယ်။

### Operator ရဲ့ အခန်းကဏ္ဍ
အေးဂျင့် တင်ပြလာတဲ့ Plan ကို သေချာ စစ်ဆေးပါ။ အဲဒီ အစီအစဉ်ဟာ လိုတာထက် ပိုပြီး ရှုပ်ထွေးနေလား (Over-engineering)၊ ၈၀/၂၀ စည်းမျဉ်းနဲ့ ကိုက်ညီရဲ့လား ဆိုတာကို သုံးသပ်ပြီး မလိုအပ်ရင် ကုဒ်တစ်ကြောင်းမှ မရေးခင် ကတည်းက ချက်ချင်း ပယ်ဖျက် ပြင်ဆင်ခိုင်းရမယ်။

---

## ၃။ State & Memory: အေးဂျင့်ရဲ့ မှတ်ဉာဏ် (Context Window & Environment)

အေးဂျင့် တစ်ယောက်ဟာ အလုပ်တစ်ခုကို ဆက်တိုက် လုပ်ဆောင်နိုင်ဖို့ သူ့မှာ မှတ်ဉာဏ်ကောင်းဖို့ လိုတယ်။ ဒီနေရာမှာ မှတ်ဉာဏ်ကို အဓိက ၂ ပိုင်း ခွဲကြည့်လို့ရတယ်:

* **State (လက်ရှိ အခြေအနေ):** မင်းရဲ့ Codebase ကြီး အခု လက်ရှိ ဘာဖြစ်နေလဲ ဆိုတာပဲ။ ဘယ်ဖိုင်တွေ ပြင်ထားလဲ၊ Git branch က ဘာလဲ၊ ခုနက Terminal မှာ Run လိုက်တာ Error တက်သွားလား၊ Success ဖြစ်လား ဆိုတဲ့ ပတ်ဝန်းကျင် အခြေအနေ (Environment Snapshot) ဖြစ်တယ်။
* **Memory (Context Window):** အေးဂျင့်ရဲ့ ကာလတို မှတ်ဉာဏ် ဖြစ်တယ်။ မင်းနဲ့ အစကတည်းက ပြောခဲ့တဲ့ ညွှန်ကြားချက်တွေ၊ ဖတ်ထားတဲ့ ဖိုင်စာသားတွေ၊ Tool တွေရဲ့ output တွေ အားလုံးကို **Context Window** ဆိုတဲ့ မှတ်ဉာဏ် သေတ္တာလေးထဲမှာ ယာယီ သိမ်းဆည်းထားတာ ဖြစ်တယ်။

### Context Window ရဲ့ ကန့်သတ်ချက် (The Memory Limit)
Context Window မှာ ကန့်သတ်ချက် (Limit) ရှိတယ်။ သေတ္တာလေးထဲကို မလိုအပ်တဲ့ ဖိုင်ကြီးတွေ၊ စာကြောင်းရေ ထောင်ချီရှိတဲ့ Error log ကြီးတွေကို အစီအစဉ်မရှိ ပုံထည့်လိုက်ရင် အေးဂျင့်ရဲ့ မှတ်ဉာဏ် ပြည့်လျှံသွားပြီး အရင်က ပေးထားတဲ့ စည်းကမ်းတွေကို မေ့သွားတာ၊ ဟိုရောက်ဒီရောက် လျှောက်လုပ်တာ (Hallucination) တွေ ဖြစ်လာတတ်တယ်။

## သတိကြီးစွာ ထိန်းကျောင်းရမယ့်အချက်တွေ (Risks & Caveats)

<Warning>
**၁။ Context Overflow (မှတ်ဉာဏ် ပြည့်လျှံပြီး ကြောင်သွားတာ):**
အေးဂျင့်နဲ့ စကားပြောတဲ့အခါ မဆိုင်တဲ့ ဖိုင်တွေ၊ မလိုအပ်တဲ့ terminal output အရှည်ကြီးတွေကို prompt ထဲ သွားမထည့်ပါနဲ့။ Context window ရှင်းလင်းနေမှ အေးဂျင့်ရဲ့ ဦးနှောက်ဟာ မြန်ဆန်တိကျပြီး မင်းရဲ့ ညွှန်ကြားချက်တွေကို မမေ့မလျော့ လုပ်ဆောင်နိုင်မှာပါ။

**၂။ Tool Loop Trap (Tool တွေ သံသရာပတ်နေတာ):**
တစ်ခါတလေ အေးဂျင့်တွေဟာ ဖိုင်တစ်ခုတည်းကိုပဲ ထပ်ခါထပ်ခါ ဖွင့်ကြည့်နေတာ၊ ဒါမှမဟုတ် Error တက်နေတဲ့ command တစ်ခုတည်းကိုပဲ ပုံစံမပြောင်းဘဲ ဆက်တိုက် run နေတာမျိုး (Loop ထဲ မိနေတာ) ဖြစ်တတ်တယ်။ အဲဒီအခါ Operator ဖြစ်တဲ့ မင်းက ချက်ချင်း ဝင်တားပြီး *"ဒီနည်းလမ်း မရဘူး၊ တခြားနည်းလမ်း ပြောင်းသုံးပါ"* ဆိုပြီး လမ်းကြောင်း တည့်ပေးရမယ်။
</Warning>

## သင်ခန်းစာ checklist (Lesson Checklist)

ဒီ Chapter 3 ကို ဖတ်ပြီးတဲ့နောက် မင်းအနေနဲ့ အောက်ပါအချက်တွေကို တကယ်ပဲ နားလည်သဘောပေါက်သွားပြီလား ကိုယ့်ကိုယ်ကိုယ် ပြန်စစ်ကြည့်ပါဦး:

- [ ] Coding Agent တွေရဲ့ အဓိက Primitives ၃ ခုဖြစ်တဲ့ Tools, Planning နဲ့ Memory အလုပ်လုပ်ပုံကို နားလည်ပြီလား။
- [ ] LLM တစ်ခုတည်းနဲ့ မလုံလောက်ဘဲ File read/write နဲ့ Terminal tools တွေ ဘာကြောင့် မဖြစ်မနေ လိုအပ်တာလဲဆိုတာ သဘောပေါက်ပြီလား။
- [ ] အေးဂျင့်ကို ကုဒ်တန်းမရေးခိုင်းဘဲ Planning အရင်လုပ်ခိုင်းတာရဲ့ အကျိုးကျေးဇူးကို လက်ခံနိုင်ပြီလား။
- [ ] Context Window ပြည့်လျှံတာကို ကာကွယ်ဖို့ မလိုအပ်တဲ့ logs တွေ၊ ဖိုင်တွေကို မထည့်ဘဲ ထိန်းချုပ်တတ်ပြီလား။

အခုဆိုရင် မင်းဟာ Agentic Coding ရဲ့ အခြေခံ သဘောတရားတွေ၊ Operator စိတ်နေစိတ်ထားနဲ့ အေးဂျင့်ရဲ့ အတွင်းပိုင်း အလုပ်လုပ်ပုံ Primitives တွေကို အပြည့်အဝ နားလည်သွားပြီ ဖြစ်လို့ **Part 1: Foundations** တစ်ခုလုံး အောင်မြင်စွာ ပြီးဆုံးသွားပါပြီ။

နောက်တစ်ဆင့်မှာတော့ လက်တွေ့ ကိရိယာတွေကို တပ်ဆင်အသုံးပြုမယ့် **Part 2: Tooling & Setup** ကို ကူးပြောင်းပြီး ခေတ်သစ် GUI App တွေ၊ Desktop IDE တွေပေါ်မှာ အေးဂျင့်ကို ကိုင်တွယ်မယ့် **[Chapter 4: Using Claude via GUI Apps & Desktop IDEs](/agentic-coding/chapter-4-claude-code-gui)** ကို ဆက်လက် လေ့လာသွားကြမယ်!
